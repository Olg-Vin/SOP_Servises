<-----User Entity------->
Long userId - id пользователя в системе
String name - короткое имя пользователя
List<MessageEntity> messages - список сообщений от пользователя
? -- User status(Active/Inactive)
deprecated - String contactInfo - не требуется, так как ответ вернётся в ту же систему

Возможные действия пользователя:
Написать и отправить сообщение
Просмотр списка сообщений
Просмотр одного сообщения и ответа на него

Вариант развития: написать ответ и вступить тем самым в цепочку сообщений
Такая реализация может быть доступна через поле category вроде "ответ"
и связи с предыдущим сообщением как LinkedList


<-------Message Entity------>
Long messageId - id сообщения в системе
UserEntity user - пользователь-владелец сообщения
String category - категория сообщения (Вопрос по продукту/Предложение/Техническая проблема)
String subject - тема сообщения (Например, сломалась кнопка, не сохраняется проект)
String body - текст сообщения (Например, проект не сохраняется, возникает такая-то ошибка)
Date createdAt - дата и время отправки сообщения
String status - статус сообщения (новое/в обработке/ожидает ответа/закрыто/отклонено)
    - позволяет видеть перемещение сообщения по сервисам быть может
ReplyEntity replies - ответ, привязанный к этому сообщению

Возможные действия:
Сообщение путешествует по gRPC сервера и там обрабатывается агентом.
Варианты развития:
Изменить тип category и status на enum.
Решить с data - заполнять перед созданием объекта.


<----Reply Entity---->
Long replyId - id ответа
MessageEntity message - сообщение, к которому привязан ответ
String body - текст ответа
Date sentAt - дата и время отправки
String status - статус сообщения (прочитано/не прочитано)

Возможные действия:
Варианты развития:
status сделать enum, по умолчанию "не прочитано"

<--------Attachment Entity--------->
Вложения (Attachment) — файлы, которые могут быть прикреплены к сообщениям или ответам. Основные атрибуты:
Идентификатор вложения (AttachmentID)
Идентификатор сообщения или ответа (MessageID или ReplyID)
Имя файла (FileName)
Тип файла (FileType)
Ссылка на файл (FileURL)


<-------Agent Entity------->
Саппорт-агент (Support Agent) — это человек или автоматическая система, которая обрабатывает сообщения пользователей и отвечает на них. Основные атрибуты:

Идентификатор агента (SupportAgentID)
Имя агента (Name)
Должность или роль (Position)
Статус активности (ActiveStatus)
